Test configurations and outputs from ICICI Bank scraping validation (2025-11-25).

## Test Configurations

### test_home_loan_nav.yaml
**Status**: ✅ Working

Navigation config using JavaScript workaround for hidden menu elements. Demonstrates:
- JS action with IIFE wrapper for clicking hidden elements
- `data-cta-region` attribute selectors
- Proper wait timing after JS-triggered navigation

```bash
aishell navigate --config tests/webscraping/test_home_loan_nav.yaml --no-headless
```

### test_smart_extract.yaml
**Status**: ✅ Working

Advanced extraction config using JavaScript regex patterns to find:
- Interest rates (looks for `X% p.a.` patterns)
- Loan amounts (looks for `X Lakh/Crore` patterns)
- Tenure (looks for `X years` patterns)
- Processing fees

Useful as a template for extracting data from pages without clean CSS selectors.

### test_debug_extract.yaml
**Status**: ✅ Diagnostic Tool

Debug config that captures all text lines containing keywords like:
- rate, fee, tenure, year, eligibility, age, income, amount, lakh, crore, %

Use this to analyze what text is actually on a page before writing extraction patterns.

```bash
aishell navigate --config tests/webscraping/test_debug_extract.yaml --no-headless
```

## Key Learnings

1. **Hidden menus**: Use `js` action instead of `hover` when elements aren't visible
2. **IIFE wrapper**: JavaScript must use `(() => { ... })()` for return statements
3. **Marketing pages**: Pages with calculators/sliders don't have static data to scrape
4. **Debug first**: Use `test_debug_extract.yaml` pattern to see raw page text before writing selectors
